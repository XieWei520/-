# Web IM 右侧会话工作台与「信息平权」品牌入口设计修订

日期：2026-04-30

## 背景

当前 Web IM B 版整体方向已经接近目标，但还有两个空间与语义问题：

1. 宽屏最右侧只显示一个「会话信息」标题，信息密度过低，造成大面积空白。
2. 左上角 `WK` 图标含义不清晰，用户无法判断它是品牌、个人入口、菜单还是状态按钮。

本修订只处理这两个 UI 问题，不改变 IM 协议、消息收发、会话同步、搜索、通话或非 Web 端导航结构。

## 已确认设计决策

### 1. 右侧栏改为「会话工作台」

右侧栏不再只是单一标题「会话信息」，而是在宽屏桌面 Web 中作为固定的会话上下文工作台使用。

默认标题：

- `会话工作台`

内容从上到下排列：

1. **成员**
   - 展示当前会话成员头像或首字占位。
   - 群聊显示前几位成员和总数入口。
   - 单聊可展示对方头像、名称和在线/身份状态。
2. **置顶 / 公告 / 状态**
   - 优先展示已有会话状态，例如「LD 已解散该群聊」。
   - 有群公告时展示群公告摘要。
   - 无公告时显示轻量空态，不留空白。
3. **文件与图片**
   - 展示最近图片、最近文件入口。
   - 第一阶段可以先做入口与空态，不新增复杂文件检索业务。
4. **快捷操作**
   - 查找聊天内容。
   - 置顶 / 取消置顶。
   - 免打扰。
   - 设置或更多会话操作。

### 2. 左上角品牌块改为「信息平权」

左侧导航 rail 顶部继续保留品牌块，但可见文案改为：

```text
信息
平权
```

约束：

- 不再显示「品牌入口」等解释性文字。
- 不再显示 `WK` 缩写。
- 品牌块保持 48px 左右的稳定点击区域，适配当前 rail 宽度。
- Tooltip 和无障碍语义可以使用「信息平权」或「打开信息平权主页」，但该说明不在界面上常驻显示。

## 布局规格

### 桌面宽屏

当 Web 宽度达到桌面工作台断点时，主界面保持四区结构：

1. 左侧导航 rail。
2. 会话列表。
3. 主聊天区。
4. 右侧「会话工作台」。

右侧工作台只在有足够宽度时常驻展示。宽度不足时应隐藏或改为抽屉，不压缩主聊天阅读空间。

### 右侧工作台尺寸

建议宽度维持在 280px 至 320px 区间：

- 过窄会导致「文件与图片」「快捷操作」拥挤。
- 过宽会挤压主聊天区。
- 当前设计稿按约 304px 表现。

### 左侧品牌块尺寸

建议：

- rail 宽度：64px 至 78px。
- 品牌块：48px x 48px。
- 品牌文字两行居中：第一行「信息」，第二行「平权」。
- 字号约 12px 至 13px，字重 800/900。

## 视觉规则

### 颜色

沿用 Web B 暖色体系：

- 页面背景：`#FFFAF5`
- 表面：`#FFFFFF`
- 弱强调背景：`#FFF7ED`
- 边框：`#FED7AA`
- 主行动色：`#F97316`
- 主行动浅色：`#FFEDD5`
- 正文：`#172033`
- 辅助文字：`#64748B`

品牌块可继续使用深橙色底，以保证左上角识别度。

### 信息密度

右侧工作台必须避免只有一个标题的空状态。即使数据不足，也需要展示结构化空态，例如：

- 成员：头像或成员数量。
- 公告：暂无群公告。
- 文件：暂无最近文件。
- 快捷操作：仍保留可用入口。

### 文案

界面可见文案：

- 左上品牌：`信息平权`
- 右侧标题：`会话工作台`
- 分区标题：`成员`、`置顶 / 公告`、`文件与图片`、`快捷操作`

不显示：

- `WK`
- `品牌入口`

## 交互规则

### 左上品牌块

第一阶段交互建议保持轻量：

- 点击品牌块回到主聊天页或当前默认工作台首页。
- 如果当前产品没有明确主页，则点击可保持当前页，但必须有 hover/pressed/focus 反馈。
- Tooltip 只在 hover 时出现，不常驻显示。

### 右侧工作台

右侧工作台中的快捷操作必须复用现有业务入口，不新增未实现的功能承诺：

- 查找：复用当前聊天搜索入口。
- 设置：复用当前更多菜单或会话设置入口。
- 置顶、免打扰：只有现有数据与动作可用时才作为可点击项；否则以禁用或隐藏处理。
- 文件与图片：第一阶段可作为视觉入口或空态；不要假造数据。

## 响应式规则

| 宽度 | 行为 |
| --- | --- |
| `< 720px` | 保持移动 Web 单列，不显示桌面右侧工作台。 |
| `720px - 1023px` | 保持列表/聊天切换或两列过渡，不显示常驻右侧工作台。 |
| `>= 1024px` | 显示桌面 rail + 会话列表 + 主聊天区。 |
| `>= 1280px` | 可常驻显示右侧「会话工作台」。 |

如果 1280px 以上仍因浏览器缩放导致空间不足，优先保证主聊天区，右侧工作台可以折叠。

## 实施边界

优先修改范围：

1. `lib/widgets/wk_tab_shell.dart`
   - 将左上 `WK` 品牌块替换为两行「信息 / 平权」。
   - 去掉可见的说明文字。
   - 保留 Tooltip / Semantics。
2. `lib/modules/conversation/web_conversation_workspace.dart`
   - 将 `_ConversationContextPanel` 从单标题升级为「会话工作台」结构。
   - 根据当前选中会话传入必要展示数据。
3. `lib/widgets/wk_web_ui_tokens.dart`
   - 如有需要，补充右侧工作台尺寸、品牌块尺寸或间距常量。
4. 相关 widget tests
   - 覆盖左上品牌文字。
   - 覆盖不再出现 `WK` 和「品牌入口」。
   - 覆盖右侧工作台标题和核心分区。

禁止范围：

- 不改消息协议。
- 不改发送、撤回、回复、转发、表情反应、通话信令逻辑。
- 不新增 AI 摘要或复杂搜索能力。
- 不为了填充右侧栏假造业务数据。

## 验收标准

### 视觉验收

1. 1440px 宽屏下右侧不再只有「会话信息」一个标题。
2. 右侧显示「会话工作台」及成员、置顶/公告、文件与图片、快捷操作分区。
3. 左上角显示「信息平权」，不显示 `WK`。
4. 左上角不显示「品牌入口」等解释性文字。
5. 主聊天区不被右侧工作台挤压到不可读。

### 功能验收

1. 点击会话后聊天仍正常打开。
2. 会话列表选择态仍正常。
3. 消息发送、撤回、回复、表情反应、通话入口不回归。
4. 非 Web 或窄屏 Web 不被强制改成桌面右侧工作台布局。

### 测试验收

至少新增或更新以下测试：

1. `WKTabShell` 桌面 rail 测试：出现「信息平权」，不出现 `WK`。
2. `WebConversationWorkspaceScaffold` 测试：宽屏右侧出现「会话工作台」。
3. 右侧工作台分区测试：成员、置顶/公告、文件与图片、快捷操作可被找到。

## 风险与处理

1. **风险：品牌文字在 48px 方块内拥挤。**
   - 处理：两行显示「信息 / 平权」，必要时降低字号到 12px。
2. **风险：右侧工作台数据不足仍显空。**
   - 处理：结构化空态必须保留，不显示空白面板。
3. **风险：右侧常驻压缩聊天区。**
   - 处理：仅在宽屏展示，空间不足时隐藏或折叠右侧。
4. **风险：当前工作区已有大量未提交改动。**
   - 处理：实现时只修改本修订涉及文件，不碰无关改动。

## 最终结论

采用用户确认版本：

- 右侧栏：升级为「会话工作台」。
- 左上角：显示品牌「信息平权」，不显示 `WK`，不显示「品牌入口」说明文字。
- 本修订只优化空间利用和入口语义，不改变核心 IM 业务行为。
