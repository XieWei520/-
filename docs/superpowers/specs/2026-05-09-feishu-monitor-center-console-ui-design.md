# Feishu Monitor Center Console UI Design

## Goal

将 WuKongIM 里的“飞书信息监控中心”从当前的状态诊断页升级成一个完整的运维控制台，吸收旧飞书转发工具里的核心页面能力，同时保持 WuKongIM 自身的管理后台风格。

目标不是 1:1 复刻旧工具，而是保留它的信息密度和实用功能，并改造成更适合长期无人值守运维的结构。

## Source Reference

用户提供的旧工具界面包含这些核心页面：

- 运行日志：黑底实时日志窗口，支持成功/错误过滤、清空日志、导出日志、自动滚动。
- 转发规则：表格展示启用状态、规则名称、来源群组、目标类型、Webhook、编辑/删除操作。
- 群组列表：表格展示群类型、群名称、群 ID、成员数、更新时间。
- 水印设置：文字转图片、图片水印、阈值、字体、透明度、位置等。
- 全局设置、账号状态、授权信息、清理图片等辅助页面。

这些能力都应该进入 WuKongIM 的飞书信息监控中心，但布局方式需要更贴合 WuKongIM。

## Recommended Layout

采用“顶部状态总览 + 内部标签页”的控制台布局，不采用旧工具那种独立软件左侧大菜单。

页面结构：

```text
飞书信息监控中心

[状态总览条]
  壳程序在线 / 飞书已登录 / 监听状态 / 今日捕获 / 今日成功 / 今日失败

[快捷操作]
  启动监控 / 停止监控 / 重新加载飞书 / 转发最近事件

[Tabs]
  运行日志 | 转发规则 | 飞书群组 | 图片处理 | 系统设置
```

### Why Tabs Instead Of Left Menu

旧工具是独立桌面软件，左侧菜单合理；WuKongIM 里这个页面只是系统管理的一部分，如果再嵌一个完整左侧菜单，会形成“菜单套菜单”，视觉层级会重。

标签页更适合：

- 管理员一眼看到运行状态。
- 在一个页面里快速切换日志、规则、群组和设置。
- 保留旧工具功能，但不让 WuKongIM 像外挂程序。
- 后续扩展 webhook 投递、失败重试、图片处理时不会挤乱主导航。

## Page Sections

### 1. Status Overview

第一屏顶部保留状态总览，采用紧凑指标卡：

- 壳程序：在线 / 离线
- 飞书账号：已登录 / 需扫码
- 监听状态：运行中 / 停止 / 异常
- 今日捕获：数字
- 今日转发成功：数字
- 今日转发失败：数字
- 最近错误：有错误时显示一行摘要

状态色：

- 绿色：正常
- 橙色：需要关注，例如未登录、暂停监听
- 红色：错误，例如 shell 不可达、投递失败

### 2. Quick Actions

状态总览下方放一行常用操作：

- 启动监控
- 停止监控
- 重新加载飞书
- 转发最近事件
- 清空运行日志

这些操作是高频运维动作，应保持在标签页之外，避免进入不同标签后找不到。

### 3. Runtime Logs Tab

对应旧工具的“运行日志”页，但做得更结构化。

布局：

- 顶部过滤条：
  - 全部
  - 成功
  - 错误
  - 捕获
  - 转发
  - 图片处理
- 右侧操作：
  - 清空日志
  - 导出日志
  - 自动滚动开关
- 主体：
  - 黑底终端样式日志窗口
  - 每行包含时间、等级、来源、消息

日志颜色：

- 成功：绿色
- 错误：红色
- 警告：橙色
- 普通信息：浅灰

### 4. Forwarding Rules Tab

对应旧工具的“转发规则”页。

表格字段：

- 启用
- 规则名称
- 来源飞书群
- 目标悟空IM群
- 目标方式
- 最近转发时间
- 今日成功
- 今日失败
- 操作

操作：

- 新增规则
- 编辑
- 删除
- 测试转发
- 批量导入
- 下载模板

MVP 可以先支持单目标群号，后续扩展为多规则表格。

### 5. Feishu Groups Tab

对应旧工具的“群组列表”页。

表格字段：

- 序号
- 聊天类型：群聊 / 单聊 / 机器人
- 群名称
- 飞书群 ID
- 成员数
- 最近消息
- 更新时间
- 转发状态

操作：

- 刷新列表
- 搜索群名或群 ID
- 为该群创建转发规则

数据来源：

- 当前 shell 的 `observed_conversations`
- 当前 shell 的 `recent_events`
- 后续如果 shell 增加更完整的群列表接口，再接入完整列表

### 6. Image Processing Tab

对应旧工具的“水印设置”和“清理图片”能力。

分为两个面板：

#### 文字转图片

- 是否启用
- 触发几率
- 图片宽度
- 字体大小
- 预览区域

#### 图片水印

- 是否启用
- 触发几率
- 水印文字
- 水印位置
- 字体字号
- 透明度
- 保存设置
- 重置默认

说明文案应简短，避免大段帮助文字压低操作效率。

### 7. System Settings Tab

系统设置聚合低频配置：

- shell 地址：默认 `http://127.0.0.1:18766`
- token
- 自动刷新间隔
- 自动转发开关
- 去重窗口
- 失败重试次数
- 掉线提醒
- 清理本地缓存

这里不放高频转发规则，避免和规则页职责重叠。

## Interaction Rules

### Manual Forwarding

管理员点击“转发最近事件”时：

1. 读取当前 `recent_events`
2. 按当前启用规则过滤
3. 对未转发过的事件生成文本
4. 投递到目标悟空IM群
5. 显示本次结果：成功、跳过、失败

### Auto Forwarding

启用自动转发后：

- 按设置间隔刷新 shell 状态
- 使用事件 `dedupe_key` 防止重复转发
- shell 离线时不反复弹错误，只在状态区提示
- 投递失败进入日志，不阻塞后续事件

### Rule Creation

从飞书群组页点击“创建规则”时：

- 自动带入来源群名和群 ID
- 目标悟空IM群需要用户选择或输入
- 保存后进入转发规则表格

## Visual Style

整体风格应保持 WuKongIM 管理中心的克制、清晰、可扫描。

设计原则：

- 避免营销式大卡片和过度装饰。
- 表格优先，适合运维反复查看。
- 日志可以使用黑底终端风格，因为这是旧工具里最有效的视觉模式。
- 卡片只用于状态总览、设置分组和日志容器。
- 不做嵌套卡片。
- 按钮使用明确动词：启动监控、停止监控、测试转发、保存设置。

## Implementation Scope

第一阶段 UI 改造建议只做：

- 顶部状态总览优化
- 快捷操作行
- Tabs 容器
- 运行日志 tab 静态/本地数据展示
- 转发规则 tab 支持当前单规则
- 飞书群组 tab 使用现有观测数据
- 图片处理 tab 先做设置表单骨架
- 系统设置 tab 先展示当前 shell 地址和自动转发配置

暂不做：

- 完整云端规则持久化
- 多目标批量投递
- 真实图片水印处理管线
- 完整权限系统
- 多账号飞书监控

## Success Criteria

- 管理员进入页面第一屏能判断：飞书是否登录、shell 是否在线、监听是否正常、今日是否有转发失败。
- 管理员能在 2 次点击内看到实时日志。
- 管理员能在转发规则页看到来源飞书群到目标悟空IM群的映射。
- 管理员能在飞书群组页确认当前 shell 已发现哪些群。
- 当前已实现的本地转发 MVP 不丢失。
- 后续 webhook/服务端投递方案可以自然替换本地 SDK 发送，不需要重做 UI 信息架构。

## Open Decision

唯一需要用户确认的产品方向：

是否采用“顶部状态总览 + 内部标签页”作为最终布局，而不是复刻旧工具左侧菜单。

推荐选择：采用顶部状态总览 + 内部标签页。
